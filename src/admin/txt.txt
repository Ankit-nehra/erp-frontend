I am building a School ERP Admin Panel + Teacher Panel using React.js frontend, Node.js + Express backend and PostgreSQL database.

Continue from the current development state. Follow the same coding style, folder structure, naming conventions, API patterns and approach we have been using.

Current stack:
- Frontend: React.js
- Styling: Tailwind CSS (No Bootstrap)
- Backend: Node.js + Express
- Database: PostgreSQL (Supabase connection)
- Database access pattern:

Example model:

import { getPool } from "../../config/postgressSupabaseDb.js";

export const functionName = async () => {

    const pool = getPool();

    const result = await pool.query(
        `
        SQL QUERY
        `
    );

    return result.rows;

};


Backend pattern:
- Separate model files
- Separate controller files
- Separate route files
- Controller handles validation, status codes and errors
- PostgreSQL duplicate error handled using code "23505"
- Frontend consumes APIs using axios
- Frontend pages are single React files with Tailwind CSS
- No edit option unless specifically required
- Admin pages generally have:
    - Add form
    - List table
    - Delete button
    - API integration

6. Teacher Assigned Class


Main mapping table:


CREATE TABLE "teacherAsignedClass" (

 id SERIAL PRIMARY KEY,

 teacher_number VARCHAR(30),

 class_id INTEGER,

 section_id INTEGER,

 subject_id INTEGER,

 session_id INTEGER,

 FOREIGN KEY (teacher_number)
